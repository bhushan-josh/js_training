# Regular Functions vs Arrow Functions

## 1. Hoisting Behavior

  ### - Regular functions are hoisted to the top. And you can access and call them even before they are declare
  ### - Arrow functions, on the other hand, cannot be accessed before they are initialized.

```javascript
    regularFunc();                                      // This is a regular function.

    function regularFunc() {
      console.log("This is a regular function.");
    }


   arrowFunc()                                      // ReferenceError: Cannot access before initialization

    const arrowFunc = () => {
      console.log("This is an arrow function.")
    }
```

## 2. Duplicate Named Parameters

  ### - When a regular function has duplicate names in the parameters, the last parameter with the duplicate name will take precedence.
  ### - Arrow functions don't allow the same parameter name, Doing so will result in a syntax error.

```javascript
    function exampleFunction(a, b, a) {
      console.log(a, b)
    }
    exampleFunction("first", "second", "third")           //third, second


    const exampleFunction = (a, b, a) => {
      console.log(a, b)
    }
    exampleFunction("first", "second", "third")           // SyntaxError: Duplicate parameter name not allowed 
```

## 3. Argument object

  ### - You can access all the arguments passed to a regular function using the arguments object. The arguments object is an array-like object that holds all the arguments passed to the function.
  ### - The arguments object is not available in arrow functions. If you try to access it in arrow functions, JavaScript will throw a reference error.

```javascript
    function logNumbers(num1, num2) {
      console.log(arguments)
    }
    logNumbers(8, 24)                                     // [Arguments] { '0': 8, '1': 24 }             


    const logNumbers = (num1, num2) => {
      console.log(arguments)
    }
    logNumbers(8, 24)                                     // Reference Error: arguments is not defined
```

## 4. Functions as Constructors

  ### - For regular functions, you can create a new instance using the new keyword. And this sets the this value to the new instance you've created.
  ### - For arrow functions, you cannot use them as constructors

```javascript
    function RegularFuncBird(name, color) {
      this.name = name
      this.species = color
      console.log(this)
    }  
    new RegularFuncBird("Parrot", "blue")                  // RegularFuncBird { name: 'Parrot', species: 'blue' }


    const ArrowFuncBird = (name, color) => {
      this.name = name
      this.color = color
      console.log(this)
    }
    new ArrowFuncBird("Parrot", "blue")                     //TypeError: ArrowFuncBird is not a constructor
```
